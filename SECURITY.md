# Security Policy

## Reporting Security Issues

If you find security concerns in our designs or documentation, please report them.

### How to Report

1. **DO NOT** create a public issue
2. Use GitHub's private vulnerability reporting
3. Or contact the maintainer directly

### What to Report

- Architectural designs with security flaws
- Missing security considerations
- Links to malicious resources
- Content promoting insecure patterns

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 5 days
- **Update**: Fix or improve content

## Security in System Design

All contributed designs should consider:
- Authentication & Authorization
- Data encryption (transit & rest)
- Input validation
- Rate limiting
- Secure storage practices

Thank you for promoting secure architectures!
